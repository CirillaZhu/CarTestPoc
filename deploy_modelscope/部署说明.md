# 部署到 ModelScope 创空间 · 操作手册

部署包就是本文件夹（`deploy_modelscope/`），已自包含、约 3.6 MB。
DeepSeek key 通过创空间**环境变量**提供（不写进代码），访客打开即用、无需输入。

---

## 一、创建创空间
1. 登录 https://www.modelscope.cn ，右上头像旁「+ 创建」→ 选「**创空间 / Studio**」。
2. 填写：英文名（如 `car-standard-qa`）、中文名、许可证（Apache 2.0）、
   公开性（公开 / 私有）、SDK 选 **Gradio**。
3. 创建后会得到仓库地址，形如：
   `https://www.modelscope.cn/studios/<你的用户名>/car-standard-qa.git`

## 二、拿访问令牌
个人中心 → 访问令牌（Access Token）→ 复制（git 推送时当密码用）。

## 三、推送代码（在本机命令行执行）
```bash
# 1) 安装 git-lfs（处理图片/向量等二进制，只需一次）
git lfs install

# 2) 克隆空仓库（把 <用户名>/<名> 换成你的）
git clone https://oauth2:<访问令牌>@www.modelscope.cn/studios/<用户名>/car-standard-qa.git
cd car-standard-qa

# 3) 把本部署包所有文件拷进去
#    （Windows 资源管理器里把 deploy_modelscope 内的全部文件复制到这个克隆目录）

# 4) 提交并推送
git add .
git commit -m "init: 汽车标准 RAG 问答"
git push
```

## 四、配置 DeepSeek key（环境变量，关键一步）
创空间「**设置 → 环境变量 / 密钥**」新增一条：
```
DEEPSEEK_API_KEY = sk-你的key
```
这样访客打开页面**直接就能问答、不用输入 key**，而且 key 不出现在公开代码里。
> ⚠️ 不要把 key 写进 app.py 或任何会被 git 提交的文件——公开创空间的代码所有人可见，等于泄露 key。

## 五、等待构建并访问
推送后创空间会自动装 `requirements.txt`、首次启动从 ModelScope 拉 bge 模型（约 1 分钟），
随后页面变「运行中」，点开就是问答界面。把这个 URL 发给别人即可远程访问。

---

## 备注
- **成本/安全提醒**：配了环境变量 key 后，**所有访客的提问都算你的 DeepSeek 账单**。
  公开创空间 = 任何人都能用、都花你的钱。建议二选一控制风险：
  - 创建时选「**私有**」，设置里邀请特定成员；或
  - 给 DeepSeek key 设较低的额度上限。
- 包内文件清单：`app.py`、`rag_core.py`、`requirements.txt`、`README.md`、
  `index/`（检索索引）、`media/`（34 张图）。
